This dashboard provides comprehensive monitoring of RabbitMQ message broker performance and queue management, offering detailed visibility into message publishing patterns, delivery rates, acknowledgment status, and queue health. This dashboard works with metrics provided by the [OpenTelemetry RabbitMQ receiver](https://github.com/open-telemetry/opentelemetry-collector-contrib/blob/main/receiver/rabbitmqreceiver).

## What This Dashboard Monitors

This dashboard tracks essential RabbitMQ metrics to help you:

- **Monitor Message Flow**: Track message publishing, delivery, and acknowledgment patterns
- **Analyze Queue Performance**: Monitor queue-specific metrics and bottlenecks
- **Track Message States**: Monitor ready, unacknowledged, and processed message counts
- **Consumer Management**: Track consumer counts and queue consumption patterns
- **Queue Health**: Monitor queue status and identify problematic queues
- **Performance Optimization**: Identify message flow bottlenecks and optimize throughput

## Metrics Included

### Overview Metrics
- **Queues Overview**: Comprehensive table showing published messages, unacknowledged messages, ready messages, and consumer counts by queue name

### Queue Metrics
- **Queue Published Messages (top 20)**: Shows message publishing rate for the top 20 most active queues over time
- **Published Messages (all)**: Shows total published message counts by queue name
- **Messages Delivered**: Shows message delivery rate grouped by queue name over time
- **Messages Acknowledged**: Shows message acknowledgment rate grouped by queue name over time

### Message State Metrics
- **Unack Messages By Queue**: Shows unacknowledged message count by queue over time
- **Current Unack Messages By Queue**: Shows current count of unacknowledged messages by queue name
