# Kubernetes Node Metrics - Detailed Dashboard

This dashboard provides detailed monitoring of Kubernetes node-level metrics, offering comprehensive visibility into CPU usage, memory utilization, filesystem usage, and network I/O performance across all nodes in your cluster.

## Dashboard Preview

![Kubernetes Node Metrics Dashboard](/img/docs/dashboards/kubernetes-node-metrics-detailed.png)

## What This Dashboard Monitors

This dashboard tracks essential Kubernetes node metrics to help you:

- **Monitor Resource Utilization**: Track CPU and memory usage across cluster nodes
- **Filesystem Management**: Monitor disk usage, available space, and capacity planning
- **Network Performance**: Observe network I/O, errors, and interface-level metrics
- **Resource Allocation**: View allocatable resources and current utilization
- **Capacity Planning**: Identify nodes approaching resource limits

## Metrics Included

### CPU Metrics
- **Node CPU Usage**: Real-time CPU utilization percentage for each node
- **CPU Allocatable**: Total CPU cores available for pod scheduling
- **CPU Usage Trends**: Historical CPU usage patterns for capacity planning

### Memory Metrics
- **Memory Usage**: Current memory consumption per node
- **Memory Available**: Available memory for new workloads
- **Memory Working Set**: Memory actively being used by processes
- **Memory RSS**: Resident set size memory usage
- **Memory Allocatable**: Total memory available for pod allocation

### Filesystem Metrics
- **Filesystem Usage**: Current disk space consumption
- **Filesystem Available**: Available disk space for new data
- **Filesystem Capacity**: Total disk capacity per node
- **Filesystem Usage Trends**: Historical storage utilization

### Network Metrics
- **Network I/O**: Ingress and egress network traffic per interface
- **Network Errors**: Network error rates by interface and direction
- **Interface Performance**: Per-interface network utilization

## Dashboard Variables

This dashboard includes pre-configured variables for filtering:

- **k8s_cluster_name**: Filter by Kubernetes cluster name
- **k8s_node_name**: Filter by specific node names (supports multi-select)

## Prerequisites

To use this dashboard effectively, ensure you have:

1. **Kubernetes Cluster**: A running Kubernetes cluster with nodes to monitor
2. **SigNoz Agent**: The SigNoz agent deployed as a DaemonSet on your cluster
3. **Kubelet Metrics**: Kubelet metrics receiver configured to collect node-level metrics
4. **RBAC Permissions**: Proper service account permissions for metrics collection

### OpenTelemetry Collector Configuration

Your OpenTelemetry collector should include the following receivers:

```yaml
receivers:
  kubeletstats:
    collection_interval: 60s
    auth_type: "serviceAccount"
    endpoint: "${K8S_NODE_NAME}:10250"
    insecure_skip_verify: true
  
  hostmetrics:
    collection_interval: 60s
    scrapers:
      cpu: {}
      memory: {}
      disk: {}
      filesystem: {}
      network: {}
```

### 2. Import to SigNoz

1. Navigate to **Dashboards** in your SigNoz instance
2. Click **New Dashboard** 
3. Click the **Import** button
4. Paste the JSON content or upload the downloaded file
5. Configure the variables to match your cluster setup
6. Save the dashboard

### 3. Configure Variables

After importing, configure the dashboard variables:

- **k8s_cluster_name**: Set to your cluster name or leave as default
- **k8s_node_name**: Select specific nodes or use "All" to monitor entire cluster

## Data Sources

This dashboard works with metrics collected from:

- **Kubelet Stats Receiver**: Node-level metrics from Kubernetes kubelet
- **Host Metrics Receiver**: System-level metrics from node hosts
- **Kubernetes Cluster Receiver**: Cluster-level metadata and status

## Troubleshooting

### Common Issues

**No Data Showing**
- Verify the OpenTelemetry collector is running on all nodes
- Check that kubelet metrics are being scraped successfully
- Ensure proper RBAC permissions for the collector service account

**Missing Node Metrics**
- Confirm the kubelet stats receiver is configured correctly
- Verify the node name variable matches your actual node names
- Check collector logs for any authentication or permission errors

**Incomplete Filesystem Data**
- Ensure the host metrics receiver includes filesystem scrapers
- Verify filesystem paths are accessible to the collector
- Check for any mount point access permissions

### Verification

To verify metrics are flowing correctly, check the SigNoz metrics explorer for:
- `k8s_node_cpu_usage`
- `k8s_node_memory_usage`
- `k8s_node_filesystem_usage`
- `k8s_node_network_io`
