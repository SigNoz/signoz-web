[Temporal](https://temporal.io/) is a developer-first, open source platform that ensures your mission-critical code runs as expected. This guide will help you instrument your Temporal applications to send traces to SigNoz using OpenTelemetry.

While Temporal comes with Jaeger tracing configured by default, this guide shows you how to configure your application to use SigNoz instead, providing enhanced observability features and integration with the SigNoz platform.

## Prerequisites

- SigNoz instance (self-hosted or Cloud)
- Temporal server (self-hosted or Cloud)
- Java 11+, Go 1.19+, or Node.js 16+ depending on your application
- Temporal application with Workflows and Activities
- Temporal CLI

### OpenTelemetry Configuration

Create a `SignozUtils` class to configure OpenTelemetry tracing with SigNoz:

```java:src/main/java/io/temporal/samples/tracing/SignozUtils.java
package io.temporal.samples.tracing;

import com.google.common.base.Splitter;
import io.opentelemetry.api.OpenTelemetry;
import io.opentelemetry.api.common.Attributes;
import io.opentelemetry.api.trace.propagation.W3CTraceContextPropagator;
import io.opentelemetry.context.propagation.ContextPropagators;
import io.opentelemetry.exporter.otlp.trace.OtlpGrpcSpanExporter;
import io.opentelemetry.sdk.OpenTelemetrySdk;
import io.opentelemetry.sdk.resources.Resource;
import io.opentelemetry.sdk.trace.SdkTracerProvider;
import io.opentelemetry.sdk.trace.export.BatchSpanProcessor;
import io.opentelemetry.semconv.resource.attributes.ResourceAttributes;
import java.util.List;
import java.util.logging.Level;
import java.util.logging.Logger;

public class SignozUtils {
    private static final Logger logger = Logger.getLogger(SignozUtils.class.getName());
    private static final String DEFAULT_SIGNOZ_ENDPOINT = "http://localhost:4317";
    private static final String DEFAULT_SERVICE_NAME = "temporal-sample-signoz";

    public static OpenTelemetry getOpenTelemetry() {
        try {
            // Get endpoint and service name from environment variables or use defaults
            String endpoint = System.getenv()
                .getOrDefault("OTEL_EXPORTER_OTLP_ENDPOINT", DEFAULT_SIGNOZ_ENDPOINT);
            String serviceName = System.getenv()
                .getOrDefault("OTEL_SERVICE_NAME", DEFAULT_SERVICE_NAME);

            // Ensure endpoint has http:// prefix
            if (!endpoint.startsWith("http://") && !endpoint.startsWith("https://")) {
                endpoint = "http://" + endpoint;
            }

            logger.info("Configuring OpenTelemetry with endpoint: " + endpoint);

            // Create OTLP exporter
            OtlpGrpcSpanExporter spanExporter = OtlpGrpcSpanExporter.builder()
                .setEndpoint(endpoint)
                .build();

            // Create resource with service name
            Resource resource = Resource.create(
                Attributes.of(ResourceAttributes.SERVICE_NAME, serviceName));

            // Add additional resource attributes if provided
            String resourceAttrs = System.getenv("OTEL_RESOURCE_ATTRIBUTES");
            if (resourceAttrs != null && !resourceAttrs.isEmpty()) {
                for (String attr : Splitter.on(',').split(resourceAttrs)) {
                    List<String> keyValue = Splitter.on('=').splitToList(attr);
                    if (keyValue.size() == 2) {
                        resource = resource.merge(Resource.create(
                            Attributes.of(keyValue.get(0).trim(), keyValue.get(1).trim())));
                    }
                }
            }

            // Configure tracer provider
            SdkTracerProvider tracerProvider = SdkTracerProvider.builder()
                .addSpanProcessor(BatchSpanProcessor.builder(spanExporter).build())
                .setResource(resource)
                .build();

            // Build and return OpenTelemetry SDK
            return OpenTelemetrySdk.builder()
                .setPropagators(ContextPropagators.create(W3CTraceContextPropagator.getInstance()))
                .setTracerProvider(tracerProvider)
                .build();
        } catch (Exception e) {
            logger.log(Level.SEVERE, "Error configuring OpenTelemetry", e);
            throw e;
        }
    }
}
```

## Troubleshooting

### Common Issues

1. **Traces Not Appearing**
   - Verify endpoint configuration
   - Check application logs for OpenTelemetry initialization errors
   - Ensure W3C context propagation is properly configured

2. **Connection Issues**
   - Ensure the endpoint includes the protocol (http:// or https://)
   - For SigNoz Cloud, use the appropriate region URL
   - Check network connectivity and firewall settings

3. **Resource Attribution**
   - Service name must be set either via environment variable or code
   - Environment should be specified for proper trace categorization

For more details, see the complete implementation in the [Temporal Java samples repository](https://github.com/temporalio/samples-java/tree/main/core/src/main/java/io/temporal/samples/tracing).
