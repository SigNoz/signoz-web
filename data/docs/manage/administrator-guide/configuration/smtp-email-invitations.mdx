SigNoz sends email invitations when you add new users to a workspace. You can enable this feature by setting a few **environment variables** before starting the SigNoz container (or Helm release / Systemd unit).


## Required variables

| Variable        | Description                                 | Example value              |
| --------------- | ------------------------------------------- | -------------------------- |
| `SIGNOZ_EMAILING_SMTP_ENABLED`  | Master switch; must be `true` to send email | `true`                     |
| `SIGNOZ_EMAILING_SMTP_FROM`     | Sender email address                        | `no-reply@yourdomain.com`  |
| `SIGNOZ_EMAILING_SMTP_HOST`     | SMTP server hostname                        | `smtp.example.com`         |
| `SIGNOZ_EMAILING_SMTP_PORT`     | SMTP server port                            | `587`                      |
| `SIGNOZ_EMAILING_SMTP_USERNAME` | Username for SMTP AUTH                      | `invitebot@yourdomain.com` |
| `SIGNOZ_EMAILING_SMTP_PASSWORD` | Password for SMTP AUTH                      | `9c0a2be1a7`               |


## Optional variables

| Variable                  | Purpose                                            | Default |
| ------------------------- | -------------------------------------------------- | ------- |
| `SIGNOZ_EMAILING_SMTP_HELLO`              | EHLO/HELO domain sent to the server                | (empty) |
| `SIGNOZ_EMAILING_SMTP_TLS_ENABLED`        | Set to `true` to use STARTTLS                      | `false` |
| `SIGNOZ_EMAILING_SMTP_SKIP_VERIFY`        | Set to `true` to skip TLS certificate verification | `false` |
| `SIGNOZ_EMAILING_SMTP_TLS_CA_FILE_PATH`   | Path to custom CA bundle (container/file path)     | (empty) |
| `SIGNOZ_EMAILING_SMTP_TLS_CERT_FILE_PATH` | Client certificate for mutual TLS (optional)       | (empty) |
| `SIGNOZ_EMAILING_SMTP_TLS_KEY_FILE_PATH`  | Client key for mutual TLS (optional)               | (empty) |
| `SIGNOZ_EMAILING_SMTP_STATIC_HEADERS`     | Comma‑separated list of `key:value` pairs          | (empty) |

<Admonition type="note">Most providers (Gmail, SES, Mailgun, etc.) require STARTTLS. Set `SMTP_TLS_ENABLED=true` and keep `SMTP_SKIP_VERIFY=false` unless you use a self‑signed cert.</Admonition>


## Docker‑Compose example

```yaml
signoz:
  image: signoz/signoz:v0.84.1
  environment:
    - SIGNOZ_EMAILING_SMTP_ENABLED=true
    - SIGNOZ_EMAILING_SMTP_FROM=no-reply@yourdomain.com
    - SIGNOZ_EMAILING_SMTP_HOST=smtp.example.com
    - SIGNOZ_EMAILING_SMTP_PORT=587
    - SIGNOZ_EMAILING_SMTP_USERNAME=invitebot@yourdomain.com
    - SIGNOZ_EMAILING_SMTP_PASSWORD=9c0a2be1a7
    - SIGNOZ_EMAILING_SMTP_TLS_ENABLED=true
    - SIGNOZ_EMAILING_SMTP_HELLO=yourdomain.com
  ports:
    - "8080:8080"
```


## Kubernetes (Helm) values snippet

If you deploy with the SigNoz Helm chart, add the variables under `additionalEnvs`:

```yaml
signoz:
  adiitionalEnvs:
    SIGNOZ_EMAILING_SMTP_ENABLED: "true"
    SIGNOZ_EMAILING_SMTP_FROM: "no-reply@yourdomain.com"
    SIGNOZ_EMAILING_SMTP_HOST: "smtp.example.com"
    SIGNOZ_EMAILING_SMTP_PORT: "587"
    SIGNOZ_EMAILING_SMTP_USERNAME: "invitebot@yourdomain.com"
    SIGNOZ_EMAILING_SMTP_PASSWORD: "9c0a2be1a7"
    SIGNOZ_EMAILING_SMTP_TLS_ENABLED: "false"
```

Apply the release:

```bash
helm upgrade --install signoz signoz/signoz -f values.yaml
```

## Verifying

1. **Restart** the SigNoz container or Pod after setting the variables.
2. Go to **Settings → Users → Invite** and send a test invite.
3. Check the application logs (stdout) for entries like `email invite sent to …`.

If the invite fails, SigNoz logs a clear error containing the SMTP server response.


That’s it! Your SigNoz instance can now email invitations using the environment‑variable setup.
