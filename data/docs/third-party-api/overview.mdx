## Overview

The **API Monitoring** feature helps you track all the APIs your internal services are calling.

It uses span attributes like
- `net.peer.name`
- `http.url`
- `http.target`
to automatically derive API calls.

## Instrumentation Requirements

To enable API monitoring:

- Ensure your application is instrumented correctly.
- For every **external call**, spans must include:
  - `net.peer.name`
  - `http.url`
  - `http.target`

> If you are using manual instrumentation, make sure these attributes are added explicitly for each outbound request.
