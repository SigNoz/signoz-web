SigNoz allows users to log in with an **identity provider (IdP)**—such as Google Workspace instead of managing separate SigNoz passwords.  All SSO configuration lives under **Settings → Organization Settings → Authenticated Domains**.

## Google Workspace

Google Workspace SSO lets members of your email domain access SigNoz with their Google accounts—no extra credentials to remember.

### Availability

- **Plans:** Community Edition, Teams, Enterprise
- **Who can configure:** Google Workspace Super‑Admins or SigNoz Org Owners


### Step‑by‑step setup

#### 1. Create OAuth 2.0 credentials in Google Cloud

1. Visit the [Google Cloud Console → Credentials](https://console.cloud.google.com/apis/credentials).
2. Select or create a **project** for SigNoz.
3. Click **Create credentials → OAuth client ID → Web application**.
4. Add an **Authorized redirect URI**:

   ```
   https://${SIGNOZ_BASEURL}/api/v1/complete/google
   ```
5. Click **Create** and note the **Client ID** and **Client Secret**. You’ll paste these into SigNoz shortly.

#### 2. Add your email domain in SigNoz

1. In SigNoz, go to **Settings → Organization Settings → Authenticated Domains**.
2. Click **Add a Domain**, then enter your company domain, e.g. `your‑domain.com`.

#### 3. Enable Google Authentication

1. After the domain appears in the list, click **Configure SSO**.
2. Choose **Google Authentication**.
3. Paste the **Client ID** and **Client Secret** from Google Cloud.
4. Click **Save Settings**.

#### 4. Enforce SSO

Back in the domain list, toggle **Enforce SSO**.  From now on, anyone with an email ending in `@your‑domain.com` must sign in via Google.

#### 5.  Test

Open a private/incognito window and visit your SigNoz URL. You should be redirected to Google for login and then back to SigNoz.


### Inviting new users

SSO does not auto‑provision accounts. Org Owners still need to **invite** people from **Settings → Organization Settings → Invite Members**.


### Troubleshooting

| Symptom                   | Fix                                                                                                                               |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| *“Authentication failed”* | Check that the redirect URI exactly matches `https://${SIGNOZ_BASEURL}/api/v1/complete/google` in Google Cloud.                   |
| Stuck in a login loop     | Ensure **Enforce SSO** is enabled **and** the user’s email domain matches the authenticated domain.                               |
| Admin locked out          | Append `?password=Y` to your login URL to temporarily fall back to password auth:`https://${SIGNOZ_BASEURL}/login?password=Y` |
